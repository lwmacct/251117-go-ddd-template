# 任务列表示例

VitePress 2.0 原生支持 GitHub 风格的任务列表，可以用于项目计划、开发进度追踪等场景。

## 📋 基本语法

```markdown
- [x] 已完成的任务
- [ ] 待完成的任务
```

## 💡 高级用法

### 嵌套任务列表

主要功能开发：

- [x] 用户系统
  - [x] 注册
  - [x] 登录
  - [x] 个人资料
  - [ ] 密码重置
- [ ] 权限系统
  - [ ] 角色管理
  - [ ] 权限分配
  - [ ] 权限验证

### 结合其他 Markdown 语法

- [x] **重要任务** - 需要优先完成
- [ ] ~~已取消的任务~~ - 不再需要
- [ ] `代码相关任务` - 涉及代码修改
- [ ] [链接任务](/guide/getting-started) - 带链接

### 结合 Badge 徽章

- [x] 实现用户认证 <Badge type="tip" text="v1.0" />
- [ ] 实现权限管理 <Badge type="warning" text="v1.1" />
- [ ] 实现日志系统 <Badge type="info" text="v2.0" />

### 结合自定义容器

::: tip 📋 本周任务

- [x] 完成 PR #120
- [x] 修复 3 个 bug
- [ ] 编写 2 篇文档
- [ ] 参加技术分享会
      :::

::: warning ⚠️ 紧急任务

- [ ] 修复生产环境 bug
- [ ] 数据库性能优化
- [ ] 安全漏洞修复
      :::

## 🎨 样式预览

任务列表的视觉效果：

✅ 已完成：带绿色勾选标记
☐ 待完成：空白复选框

## 📝 Markdown 源代码

想要创建任务列表，只需：

```markdown
- [x] 已完成（x 表示已选中）
- [ ] 待完成（空格表示未选中）
```

## 💡 使用建议

1. **项目管理**
   - 使用任务列表跟踪功能开发进度
   - 结合 Badge 标注版本号

2. **文档编写**
   - 列出待完善的文档章节
   - 标记已完成的部分

3. **Code Review**
   - 创建审查清单
   - 确保不遗漏重要检查项

4. **个人计划**
   - 制定每日/每周/每月计划
   - 追踪完成情况

5. **团队协作**
   - 分配任务给不同成员
   - 可视化团队进度
